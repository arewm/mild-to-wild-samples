# Medium

**Can we verify our own build and summarize the result?**

The medium level runs the same SLSA provenance checks as mild against a newly built image, then produces a Verification Summary Attestation (VSA) at SLSA Build Level 2. This decouples verification from enforcement — downstream consumers like admission controllers check the VSA instead of re-evaluating the provenance themselves.

## Supported Build Systems

The medium policies work with multiple build systems:
- **Tekton Pipelines + Chains**: Original target, uses SLSA v2alpha4 format
- **GitHub Actions**: Native SLSA v1.0 provenance with workflow attestation

## Example with GitHub Actions

The image `ghcr.io/puerco/mild-to-wild-samples:7d7c8864f71708ff051ca35dcb68d9e196034aa8` demonstrates medium security verification with GitHub Actions:

- ✅ **SLSA v1.0 provenance** from GitHub's native attestation
- ✅ **Trusted source branch** (`refs/heads/main`)
- ✅ **Known builder identity** (GitHub Actions workflow)
- ⚠️  **SBOM optional** (not generated by default in GitHub Actions)

See [conforma/](conforma/) and [ampel/](ampel/) for engine-specific policies and invocation instructions.
